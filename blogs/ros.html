What is ROS?

ROS, or Robot Operating System, is a flexible framework for writing robot software. It is a collection of tools, libraries, and conventions that aim to simplify the task of creating complex and robust robot behavior across a wide variety of robotic platforms.

Why ROS?

ROS provides a structured communications layer above the host operating systems of a heterogeneous compute cluster. ROS is language-agnostic, meaning you can use it with C++, Python, and other languages. It also has a large community and a wealth of resources, making it easier to find support and solutions to problems.

ROS Architecture

ROS Nodes

Nodes are processes that perform computation. ROS is designed to be modular at a fine-grained scale; a robot control system usually comprises many nodes. For example, one node controls a laser range-finder, one node controls the robot's wheel motors, one node performs localization, one node performs path planning, and so on.

ROS Messages and Topics

Nodes communicate with each other by passing messages. A message is a simple data structure, comprising typed fields. Standard primitive types (integer, floating point, boolean, etc.) are supported, as are arrays of primitive types. Messages are transmitted on a topic, which is a named bus over which nodes exchange messages.

ROS Services and Actions

Services in ROS allow nodes to send a request and receive a response. This is useful for tasks that are not continuous, such as querying the state of a sensor or commanding a robot to move to a specific location. Actions are similar to services but are designed for tasks that take time to complete, such as moving a robot arm to a position.

ROS Bag Recordings

ROS bags are a format for saving and playing back ROS message data. Bags are typically created by a tool like rosbag, which subscribes to one or more ROS topics, and writes the serialized message data to a file as it is received. This is useful for offline analysis and debugging.

Case Study: How we use ROS?

In Swayansaasita, we use ROS to integrate various sensors and actuators on our robotic platforms. ROS allows us to modularize our code, making it easier to develop, test, and deploy new features. We also use ROS's visualization tools to monitor the state of our robots in real-time and to debug issues as they arise.
